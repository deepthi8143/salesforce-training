# 📘 Salesforce Day 12 Notes

# ⚡ Salesforce DX

Salesforce DX is a modern development experience for Salesforce projects.

Features:
- Source-driven development
- GitHub integration
- Team collaboration
- Better deployment workflow

---

# 💻 Salesforce CLI

CLI helps developers:
- Create projects
- Deploy applications
- Run commands quickly
- Improve workflow efficiency

---

# 🛠️ Version Control

Version control helps teams:
- Track changes
- Prevent code conflicts
- Restore previous versions
- Collaborate safely

GitHub is commonly used for version control.

---

# 👥 Team Collaboration

Large enterprise systems involve:
- Multiple developers
- Continuous deployment
- Testing workflows

---

# 🏗️ Sandbox vs Production

## Sandbox
Safe environment for development and testing.

## Production
Live environment used by real users.

Changes should always be tested before production deployment.

---

# 🔄 Deployment Workflow

Typical workflow:
Development →

Testing →

GitHub commit →

Review →

Deployment →

Production

---

# ⚠️ Risks Without Workflow Management

- Code conflicts
- Deployment failures
- Data loss and Production bugs
- No rollback capability

---

# 🧠 Key Understanding

Professional software engineering is not only coding. It also requires:
- Collaboration
- Reliability
- Version control
- Deployment management
- Team coordination
