### 1️⃣ Screen Flow
- User-driven flow
- Requires user input
- Used for guided processes

### 2️⃣ Record-Triggered Flow
- Runs automatically when record is created/updated
- No user interaction needed
- Used for backend automation

# 🔄 Flow Design (Example)

## Process: Auto Update Course Seats

### Trigger
- When a student enrolls in a course

### Steps
- Fetch course record
- Check available seats
- Reduce seat count by 1

### Decision
- If seats = 0 → mark course as FULL

### Final Action
- Update course record
- Send notification if full

# ⚙️ Manual vs Automated Process

## Example: Student Enrollment

### Manual Process
- Staff checks available seats manually
- Updates Excel sheet
- Sends emails manually
- High chance of errors

---

### Problems in Manual System
- Time-consuming
- Human errors
- Delayed updates
- Poor scalability

---

### Automated Process (Salesforce Flow)
- System automatically updates records
- Sends emails instantly
- Updates seat count in real-time
- No manual intervention needed
