# 📘 Salesforce Task Day 2 Notes

# 🌐 What is Salesforce Platform?

Salesforce Platform is a cloud-based application development and CRM platform used to build business applications, manage customer relationships, automate workflows, and store data securely.

It provides tools for:
- Application development
- Data storage and management
- Security and authentication
- Automation and workflows
- Reporting and analytics
- APIs and integrations

Salesforce allows businesses to develop applications without maintaining physical infrastructure.

---

# 🏗️ Salesforce Platform Structure

The Salesforce Platform is organized using several important components:

- Apps
- Objects
- Tabs
- Records
- Fields
- Dashboards
- Reports

These components work together to create complete business applications.

---

# 📱 What is an App in Salesforce?

An App is a collection of tabs, objects, reports, dashboards, and functionalities designed for a specific business purpose.

Apps help users organize work efficiently inside Salesforce.

### Examples of Apps
- Sales App
- Service App
- Marketing App
- HR Management App

### Features of Apps
- Easy navigation
- Centralized workflow
- Access to business data
- Improved productivity

---

# 📦 What is an Object?

Objects are database tables used to store and organize information inside Salesforce.

Objects contain:
- Fields
- Records
- Relationships

---

## 🔹 Standard Objects

Standard Objects are already provided by Salesforce.

### Examples
- Account
- Contact
- Lead
- Opportunity
- Case

These objects are commonly used in CRM applications.

---

## 🔹 Custom Objects

Custom Objects are created according to business requirements.

### Examples
- Student Object
- Employee Object
- Course Object
- Inventory Object

Custom Objects allow businesses to build unique applications.

---

# 📑 What is a Tab?

Tabs are user interface components used to access objects, applications, dashboards, and reports.

Tabs improve navigation and make it easy for users to work with data.

### Examples
- Accounts Tab
- Contacts Tab
- Reports Tab
- Dashboard Tab

---

# 🗂️ Records and Fields

## Record
A Record is a single entry inside an object.

## Field
A Field stores specific information about a record.

### Example

| Name | Email | Phone |
|---|---|---|
| Rahul | rahul@gmail.com | 9876543210 |

Here:
- Rahul is a Record
- Name, Email, and Phone are Fields

---

# 🏛️ Salesforce Architecture

Salesforce uses Multi-Tenant Architecture.

---

## 🔹 Multi-Tenant Architecture

In Multi-Tenant Architecture:
- Multiple organizations use the same Salesforce platform
- Each organization’s data remains secure and isolated
- Infrastructure resources are shared efficiently
- Automatic updates are provided to all users

### Advantages
- Scalability
- Cost efficiency
- High availability
- Easy maintenance

---

# ⚙️ Configuration vs Coding

| Configuration | Coding |
|---|---|
| Uses clicks and tools | Uses programming languages |
| No coding required | Coding knowledge needed |
| Faster implementation | More customization possible |
| Used for simple requirements | Used for advanced requirements |

---

# 👨‍💻 How Developers Extend Salesforce

Salesforce Developers extend functionality using:
- Apex Programming
- Lightning Components
- APIs
- Integrations
- Automation tools

This helps organizations build custom business solutions.

---

# 👨‍💼 Salesforce Admin vs Developer

| Salesforce Admin | Salesforce Developer |
|---|---|
| Configures Salesforce | Builds custom applications |
| Uses setup tools | Writes Apex code |
| Manages users | Develops integrations |
| Creates reports | Builds automation |
