## 1️⃣ What is an App in Salesforce?

An App in Salesforce is a collection of tabs, objects, dashboards, reports, and functionalities grouped together for a specific business purpose.

Apps help users organize work and access related features in one place.

### Examples
- Sales App
- Service App
- Marketing App

---

## 2️⃣ What is an Object?

An Object is a database table used to store information in Salesforce.

Objects contain:
- Fields
- Records
- Relationships

### Examples
- Account
- Contact
- Lead
- Opportunity

---

## 3️⃣ Difference between App and Object

| App | Object |
|---|---|
| Collection of tabs and features | Database table |
| Used for workflow management | Used for storing data |
| Contains multiple objects | Contains records and fields |

---

## 4️⃣ What is Multi-Tenant Architecture?

Multi-Tenant Architecture is a cloud architecture where multiple organizations share the same Salesforce infrastructure while keeping their data secure and separate.

This architecture provides:
- Scalability
- Security
- Resource sharing
- Automatic updates

---

## 5️⃣ When should we use configuration instead of code?

Configuration should be used when business requirements can be implemented using Salesforce tools without writing code.

It is preferred because:
- Faster implementation
- Easier maintenance
- No programming required

---

## 6️⃣ How does Salesforce allow developers to extend functionality?

Salesforce allows developers to extend functionality using:
- Apex Programming
- Lightning Components
- APIs
- Integrations
- Automation tools

These technologies help developers build scalable and customized business applications.
