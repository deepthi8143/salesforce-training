# 📝 Task Answers – Day 13

## 1. What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It automates testing and deployment workflow.

---

## 2. Why deployment workflow matters?

Deployment workflow prevents:
- Production bugs
- Downtime
- Data loss
- Unstable releases

---

## 3. Problems without version control

Without GitHub:
- Code conflicts happen
- Work may get overwritten
- Rollback becomes difficult

---

## 4. Why teams use branches?

Branches allow developers to work independently without affecting production code.

---

## 5. Why testing before deployment matters?

Testing helps identify bugs before software reaches users.

---

## 6. Difference between writing code and enterprise software engineering

Writing code focuses on features.

Enterprise engineering focuses on:
- Reliability
- Scalability
- Collaboration
- Deployment
- Maintenance
