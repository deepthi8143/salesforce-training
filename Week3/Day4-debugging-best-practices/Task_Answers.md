# 📝 Task Answers – Day 16

## 1. Bug Analysis

### Duplicate Notifications

Problem:
Users receive the same notification multiple times.

Debugging Approach:
- Check Flow execution.
- Review Trigger logic.
- Verify duplicate automation rules.
- Analyze debug logs.

---

### Attendance Calculation Wrong

Problem:
Attendance percentage is incorrect.

Debugging Approach:
- Verify formula fields.
- Check calculation logic.
- Compare database values.
- Test with sample records.

---

### Flow Not Triggering

Problem:
Automation does not execute.

Debugging Approach:
- Verify Flow activation.
- Check entry conditions.
- Review Flow debug details.
- Test with sample records.

---

### Approval Process Stuck

Problem:
Approval request does not move forward.

Debugging Approach:
- Check approval configuration.
- Verify approver assignment.
- Review approval history.
- Analyze related logs.

---

## 2. Performance Thinking

### UI Issues

- Slow page loading
- Delayed dashboard updates
- Large component rendering

### Backend Issues

- Slow Apex execution
- Heavy business logic processing

### Database Issues

- Slow queries
- Large data volume

### Notification Issues

- Delayed email delivery
- Bulk notification processing delays

### Automation Issues

- Multiple Flows executing simultaneously
- Complex Trigger execution

---

## 3. Maintainability Thinking

Developers should write modular and reusable code because:

- Easier debugging
- Better scalability
- Faster development
- Improved collaboration
- Reduced maintenance effort

Quick fixes may work temporarily but create long-term problems.

---

## 4. Reflection

Debugging is one of the most important software engineering skills because enterprise applications are complex and issues can affect many users. Effective debugging helps developers identify problems quickly, improve reliability, and maintain system performance.
